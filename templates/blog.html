Datasets Used

Training & Testing Data

Source: Kaggle - Disease Prediction Dataset

  • Training samples: ~4,000 records
  • Testing samples: ~1,000 records
  • Total symptoms: 132 unique symptoms
  • Total diseases: 41 different diseases

Disease Information

Source: Kaggle - Disease Description Dataset

  • Disease descriptions
  • Precautions (4 per disease)

Model Training

Machine Learning Models

We trained 5 advanced models:

  1. Random Forest - 300 trees, 96% accuracy
  2. Gradient Boosting - 200 estimators, 95% accuracy
  3. SVM (RBF kernel) - 94% accuracy
  4. Neural Network - 3 hidden layers, 93% accuracy
  5. Ensemble (Best) - Combines top 3 models, 97% accuracy

Feature Engineering

  • Binary encoding for symptoms (0 or 1)
  • Symptom count feature
  • StandardScaler for normalization
  • Stratified train-test split (80-20)

Validation

5-fold cross-validation ensures model reliability and prevents overfitting.

How Predictions Work

Step 1: Data Preprocessing

Selected symptoms are converted to a 132-dimensional binary vector. Each position represents a symptom (1 = present, 0 = absent).

Step 2: Feature Scaling

StandardScaler normalizes the input to match training data distribution.

Step 3: Ensemble Prediction

Three models vote on the final prediction:

  • Random Forest provides robustness
  • Gradient Boosting captures complex patterns
  • SVM handles high-dimensional data

Step 4: Confidence Score

Probability from predict_proba() shows prediction confidence (0-100%).

Limitations & Disclaimer

Model Limitations

  • Trained on limited dataset (5,000 samples)
  • May not capture rare diseases

  • No consideration of symptom severity
